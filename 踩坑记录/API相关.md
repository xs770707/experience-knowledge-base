# API相关踩坑记录

### 不看文档就调API
- **场景**：猜测API端点和参数
- **坑**：猜错反复重试，浪费资源
- **教训**：先读官方文档，再动手
- **正确做法**：先读官方API文档，确认端点和参数

### API文档有但实际不可用
- **场景**：API文档说有某个功能，但调用返回401或40013
- **坑**：文档和实际不一致，反复尝试浪费时间
- **教训**：API文档可能滞后，不是写了就能用
- **正确做法**：遇到异常返回码先确认是否是平台端问题，不要反复重试，记录状态寻找替代方案

### 功能没验证就下结论
- **场景**：用户问"能不能做XX"，凭"理论上可以"就直接回答能
- **坑**：承诺了功能但实测不行，误导用户浪费时间
- **教训**：不能凭理论推断就下结论，实测才算数
- **正确做法**：先实测，再回复。做不到的事不说能做到
