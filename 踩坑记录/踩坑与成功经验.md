# 踩坑与成功经验

## API相关

### 不看文档就调API
- **发生了什么**：猜测API端点和参数，猜错反复重试
- **为什么**：没读官方文档就动手
- **更快的方法**：先读官方API文档，确认端点和参数再调用
- **还能更快吗**：目前最快
- **适用**：所有平台 / API调用

### API文档有但实际不可用
- **发生了什么**：API文档说有某个功能，但调用返回401或40013
- **为什么**：API文档滞后，不是写了就能用
- **更快的方法**：遇到异常返回码先确认是否是平台端问题，不要反复重试，记录状态找替代方案
- **还能更快吗**：目前最快
- **适用**：所有平台 / API调用

### 功能没验证就下结论
- **发生了什么**：凭"理论上可以"就直接回答能，实测不行
- **为什么**：没实测就下结论
- **更快的方法**：先实测，再回复。做不到的事不说能做到
- **还能更快吗**：目前最快
- **适用**：所有场景 / 功能确认

### GitHub API命令行上传比网页操作高效
- **发生了什么**：网页端逐个Create new file，操作繁琐且容易出错
- **为什么**：网页端不适合批量操作
- **更快的方法**：用git clone + 本地编辑 + git push批量上传
- **还能更快吗**：目前最快
- **适用**：GitHub / 网站 / 开发工具

## 浏览器操作相关

### 用浏览器做API能做的事
- **发生了什么**：模拟人类操作浏览器，效率低、成本消耗大
- **为什么**：没先查API就直接开浏览器
- **更快的方法**：API优先，浏览器兜底。底层代码跑得通绝不绕上去开浏览器
- **还能更快吗**：目前最快
- **适用**：所有平台 / API vs 浏览器选择

### 浏览器执行脚本变量冲突
- **发生了什么**：在浏览器里用eval执行JavaScript，变量持久化导致重复声明报错
- **为什么**：浏览器环境有状态残留，不像每次都是干净环境
- **更快的方法**：每次eval使用不同变量名，或关闭浏览器重新打开
- **还能更快吗**：目前最快
- **适用**：浏览器 / 脚本执行

### 界面坐标定位不准
- **发生了什么**：用坐标盲点操作界面元素，坐标偏移导致操作错误
- **为什么**：没先获取可交互元素就盲操作
- **更快的方法**：先获取交互元素引用，再用引用或语义定位操作
- **还能更快吗**：目前最快
- **适用**：浏览器 / 界面操作

## 平台适配相关

### 不查经验就动手
- **发生了什么**：执行类似任务时，没查之前的成功路径就直接开干，反复试错
- **为什么**：忘了"昨天做对的事今天也要做对"
- **更快的方法**：动手前先搜经验，看之前有没有成功路径
- **还能更快吗**：目前最快
- **适用**：所有场景

### 复杂链路可靠性低
- **发生了什么**：通过多步中转实现功能，任何一环出问题整个链路就断
- **为什么**：链路越长越不稳定
- **更快的方法**：优先使用平台原生渠道，避免多步中转
- **还能更快吗**：目前最快
- **适用**：所有平台 / 架构选择

### 子任务效率低
- **发生了什么**：简单任务也派子任务，子任务重复主任务的错误
- **为什么**：3步以内能搞定的事不派子任务
- **更快的方法**：简单任务自己做，给子任务的指令要明确方法
- **还能更快吗**：目前最快
- **适用**：任务拆分

### GitHub国内访问不稳定
- **发生了什么**：国内访问github.com页面偶发无法打开
- **为什么**：网络环境问题，不是操作问题
- **更快的方法**：先用API接口确认仓库状态，排除操作问题；网页打不开时等一段时间重试或换网络
- **还能更快吗**：目前最快
- **适用**：GitHub / 网站 / 国内网络

### GitHub Token类型选错
- **发生了什么**：Fine-grained tokens权限配置复杂，漏配导致操作失败
- **为什么**：简单上传用了复杂的token类型
- **更快的方法**：生成Personal Access Token时选Tokens (classic)，勾选repo权限即可
- **还能更快吗**：目前最快
- **适用**：GitHub / 网站 / 开发工具

## 数据存储相关

### 所有经验堆在一个文件里
- **发生了什么**：不同领域的操作经验全写在同一个文档里，查找困难
- **为什么**：没按领域建独立文档
- **更快的方法**：每涉及一个新领域或新工具，建立独立的操作指南文档
- **还能更快吗**：目前最快
- **适用**：所有场景 / 知识管理

### 依赖不稳定的外部存储
- **发生了什么**：把重要数据存在第三方平台，平台挂了数据就没了
- **为什么**：第三方服务可能下线、限流、改规则
- **更快的方法**：关键数据自包含存储，第三方只做备份不做主存
- **还能更快吗**：目前最快
- **适用**：所有场景 / 数据存储

## 效率方法

### 用快捷键列表快速了解软件全貌
- **发生了什么**：新接触一个软件，不知道能做什么
- **为什么**：没有找到完整功能清单
- **更快的方法**：查快捷键列表，比翻菜单快10倍
- **还能更快吗**：目前最快
- **适用**：本地软件 / 影音类 / 设计类 / 所有GUI软件

### 剪映v13→v12快捷键差异
- **变了什么**：v13新增"智能字幕"Ctrl+Shift+S，"曲线变速"从Ctrl+T改为Ctrl+Shift+T
- **影响**：旧版Ctrl+T功能在v13失效，需要用新快捷键
- **适用**：剪映PC版 v13 / 本地软件 / 影音类
