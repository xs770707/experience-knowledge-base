# AI执行经验知识库

一个公开的AI执行经验知识库，收集真实场景中的踩坑记录和成功路径。

## 这是什么

当你让AI干活时，有些坑会反复踩、有些成功路径会反复用。这个知识库就是把那些经验沉淀下来，让所有人（和所有AI）都能少走弯路。

## 怎么用

- **直接读**：浏览仓库里的文档，看看别人踩过什么坑
- **给AI用**：把文档拉到本地，作为AI的参考经验库
- **贡献经验**：按格式提PR，脱敏后提交

## 贡献格式

每条经验统一四要素：

```markdown
### 标题
- **场景**：
- **坑**：
- **教训**：
- **正确做法**：
```

成功路径同理：

```markdown
### 标题
- **方法**：
- **关键**：
```

## 贡献规则

1. **必须脱敏**：去掉所有业务细节、账号信息、项目名称、公司信息
2. **必须真实**：自己亲身踩过的坑或验证过的成功路径，不写"理论上应该"
3. **四要素完整**：场景、坑/方法、教训/关键、正确做法，缺一不可
4. **按领域放文件**：不知道放哪个文件就新建一个，文件名直接体现领域

## 文件结构

```
踩坑记录/
├── API相关.md        ← API调用、端点、认证等踩坑
├── 浏览器操作相关.md  ← 浏览器模拟、界面操作等踩坑
├── 平台适配相关.md    ← 跨平台、版本差异等踩坑
├── 数据存储相关.md    ← 文件存储、数据同步等踩坑
└── ...按领域自生长

平台操作指南/
├── 各平台独立文档
└── ...
```
